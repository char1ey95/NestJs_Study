># 로그인 섹션

- 지갑생성(회원가입), 잠금해제(로그인) 기능이 있는 파트

<br />


## 기능
      i) 지갑 생성
        - 니모닉 가져오기
        - 니모닉 입력하기
        - 비밀번호 입력하기(생성)

     ii) 지갑 가져오기
        - 니모닉 입력하기
        - 비밀번호 입력하기(생성)

    iii) 잠금 해제
        - 비밀번호 입력하기(잠금해제)


<br />

---

## `니모닉 가져오기`(지갑 생성)

- 지갑 생성하기 버튼 클릭시 이동하며, 백서버에서 니모닉을 가져와야한다.
- 가져온 니모닉은 다음 페이지(니모닉 입력)에서 검증을 위해 상태에 저장한다.

```json
// input
{}

// output
{
    "mnemonic": [ "", "", "", "", "", "", "", "", "", "", "", "" ]
}
```

<br />

---

## `니모닉 입력하기`(지갑생성 + 지갑 가져오기)

- 계정 가져오기 페이지에서는 입력받은 니모닉을 이용해 계정을 생성(요청)해준다.
- 계정 생성 페이지의 경우 입력받은 니모닉과 이전 페이지에서 생성된 니모닉과 일치여부를 판별 후에 계정을 생성(요청)한다.

```json
// input
{
    "mnemonic": [ "", "", "", "", "", "", "", "", "", "", "", "" ]
}

// output
{
    "account":{
        "privateKey":"",
        "publicKey":"",
        "address":""
    }
}
```

---

## `비밀번호 입력하기`(지갑생성 + 지갑 가져오기)

- 이 페이지에서는 닉네임, 비밀번호를 입력받는 페이지이다.
    - 입력받은 닉네임의 경우, 그대로 로컬 스토리지에 계정과 함께 저장한다.
    - 입력받은 비밀번호는 로컬 스토리지에 저장을 위해 (혹은 DB에 넣을지 결정) 백서버에 보내 암호화를 진행한다.
- 비공개키(privateKey)의 암호화 진행을 위해 백서버에 넘겨준다.

<br />

### ※.고려사항

    DB에 저장할지, 로컬에 저장할지 확실하게 결정이 필요


```json
// DB에 비밀번호와 게정을 저장하는 경우, 닉네임의 경우는 선택
// input
{
    "address":"",
    "password":"",
    "privateKey":"",
    "nickname":""
}

// output
// 암호화 값
{
    "password":"",
    "privatekey":""
}

//=====================================================

// 저장하지 않는 경우
// input
{
    "password":"",
    "privateKey":""
}

// output
// 암호화 값
{
    "password":"",
    "privatekey":""
}
```

## `비밀번호 입력하기`(잠금해제)

<br />

### ※.고려사항

```js
// 1. 닉네임, 프로필 이미지 url의 저장위치

1-1. 닉네임과 프로필 url을 로컬 스토리지에 모든 게정에 대해 가지고 있는다.

1-2. 닉네임과 프로필 url을 DB서버에 account와 함께 저장해 불러온다.

1-3. 닉네임과 프로필 url을 DB와 로컬 모두에 저장해놓는다.

// 2. 비밀번호의 비밀번호 검증 진행할 곳

2-1. 프론트에서 검증할 경우
    - 오프라인 로그인 가능
    - 닉네임과 프로필 정보를 로컬에 반드시 가지고 있어야한다.

2-2. 백에서 검증할 경우
    - 오프라인 로그인 불가능
```

만약, 서버에 비밀번호가 일치하는지 알아보기 위해서는 비밀번호의 검증 로직이 필요하다.

```json
// input
{
    "password":""
}

// output
{
    "success": true
}
```

서버에 요청하지 않는 경우, 프론트에서 검증로직(암호화)이 필요하다.

># 메인(자산) 섹션

- 선택된 계정의 지갑주소에 따라 해당하는 자산을 보여주는 페이지

## 기능

      i) 잔액 정보 불러오기
        - 내가 가진 자산(통화, 토큰)들의 잔액 불러오기
            - 네트워크의 화페(통화) 잔액(balance) 표시하기(usd, krw)
            - 각 토큰의 CA를 이용해서 잔액을 불러와야함

     ii) 커스텀 토큰 추가하기
        - 토큰이 ERC20의 기준에 맞는지 검증한 뒤 추가하기

    iii) 내가 가진 NFT를 불러오기
        - 메인페이지에서 보여주기(리스트)
        - 프로필에서 보여주기(리스트)
        - 각 NFT에 대해서 보여주기(단일 개체)

     iv) 트랜잭션 기능
        - 송금하기
        - 트랙잭션 내용(거래내역) 불러오기

<br />

---

## `잔액 불러오기 - 통화만`(잔액 정보 불러오기)

- 네트워크의 화페(통화) 잔액(balance) 표시하기(usd, krw)

<br />

### ※.고려사항

    통화의 잔액을 불러올떄 토큰과 함꼐 불러올지 결정해야함

    통화가 뭔지 알기위해 네트워크 식별자를 보내야할지 결정해야함

    토큰의 CA값을 가지고 있을지 결정해야함

    가져올 통화 및 토큰의 속성(ca, id, name, symbol 등등) 정의필요

<br />

```json
// input
{
    "address":""
}

// output
{
    "id":"",
    "symbol":"",
    "balance": 0,
}
```

## `잔액 불러오기 - 통화와 토큰`(잔액 정보 불러오기)

- 내가 가진 토큰들의 잔액 불러오기 잔액(balance) 표시하기(usd, krw)

```json
// input
{
    "address":"",
    "myToken": [
        "ca1",
        "ca2",
        "ca3",
        ...
    ]
}
```

<br />

---

## `토큰 추가`(커스텀 토큰 추가하기)

- ERC20의 검증을 진행하고 내 토큰 목록 myToken 혹은 tokenList에 추가

## 고려사항

    토큰 이미지를 어떻게 처리할지 결정해야함(학습 필요?)

```json
// input
{
    "contractAddress":"",
    "symbol":"",
    "decimal":0
}

// output
// erc-20 토큰의 검증만 진행
{
    "success": true,?
}
```

    "address":""

    "transactionData": [{},{},{},...]
}

># 코인 정보 섹션

- 코인 정보를 보여주는 페이지
- 대부분의 정보는 코인게코, 코인 마켓캡을 통해서 가져오기

## 기능

   i) 코인 랭킹 가져오기
  ii) 한개의 코인에 대한 자세한 정보 가져오기

### `코인 랭킹 가져오기`(코인 랭킹 가져오기)

```json
// input
// 식별자를 이용해 페이지 네이션 구현을 위함
{
    "index":0
}

// output
{
    "coinInfomation":[
        {
            "id":"",
            "symbol":"",
            "current_price":0,
            "change_percent":0
        },
        {            
            "id":"",
            "symbol":"",
            "current_price":0,
            "change_percent":0
        },
        {           
            "id":"",
            "symbol":"",
            "current_price":0,
            "change_percent":0
        }
    ]
}
```
