# Login

- 로그인 로직페이지

## getMnemonic(지갑 생성 버튼 클릭시 니모닉 가져오기)

- create new wallet 버튼 클릭시 이동하며, 서버에서 니모닉을 가져와야한다.
- 가져온 니모닉은 다음 페이지(니모닉 입력)에서 검증을 위해 상태에 저장한다.

```json
// input
{}

// output
{
    "mnemonic": [ "", "", "", "", "", "", "", "", "", "", "", "" ]
}
```

## postMnemonic(니모닉 입력 => 계정 생성)

- 계정 가져오기 페이지에서는 입력받은 니모닉을 이용해 그냥 계정을 생성(요청)해준다.
- 계정 생성 페이지의 경우 입력받은 니모닉과 이전 페이지에서 생성된 니모닉과 일치여부를 판별 후에 계정을 생성(요청)한다.

```json
// input
{
    "mnemonic": [ "", "", "", "", "", "", "", "", "", "", "", "" ]
}

// output
{
    "account":{
        "privateKey":"",
        "publicKey":"",
        "address":""
    }
}
```

## postJoin?(닉네임, 비밀번호 입력)

- 이 페이지에서는 닉네임, 비밀번호를 입력받는 페이지이다.
    - 입력받은 닉네임의 경우, 그대로 로컬 스토리지에 계정과 함께 저장한다.
    - 입력받은 비밀번호는 로컬 스토리지에 저장을 위해 (혹은 DB에 넣을지 결정) 백서버에 보내 암호화를 진행한다.
- 비공개키(privateKey)의 암호화 진행을 위해 백서버에 넘겨준다.

```json
// DB에 비밀번호와 게정을 저장하는 경우, 닉네임의 경우는 선택
// input
{
    "address":"",
    "password":"",
    "privateKey":"",
    "nickname":""
}

// output
// 암호화 값
{
    "password":"",
    "privatekey":""
}

//=====================================================

// 저장하지 않는 경우
// input
{
    "password":"",
    "privateKey":""
}

// output
// 암호화 값
{
    "password":"",
    "privatekey":""
}
```

# Lock(이슈사항)

`!이슈사항`
1. 닉네임, 프로필 이미지 url의 저장위치

      i. 닉네임과 프로필 url을 로컬 스토리지에 모든 게정에 대해 가지고 있는다.
     ii. 닉네임과 프로필 url을 DB서버에 account와 함께 저장해 불러온다.
    iii. 닉네임과 프로필 url을 DB와 로컬 모두에 저장해놓는다.

2. 비밀번호의 비밀번호 검증 진행할 곳

     i. 프론트에서 검증할 경우
        - 오프라인 로그인 가능
        - 닉네임과 프로필 정보를 로컬에 반드시 가지고 있어야한다.
    ii. 백에서 검증할 경우
        - 오프라인 로그인 불가능

## 잠금화면

만약, 서버에 비밀번호가 일치하는지 알아보기 위해서는 비밀번호의 검증 로직이 필요하다.

## 자산 정보 불러오기

### Token(토큰 잔액 불러오기)

- 네트워크의 화페(통화) 잔액(balance) 표시하기(usd, krw)
- 아래에 있는 토큰 정보들과 함께 보내는 것이 좋을거 같아, 사용하지 않을 가능성이 있음
- 토큰들의 CA값을 가져와서 가지고 있을지 생각해보기...

```json
// input
// 네트워크 식별자, 어떤 네트워크에 연결되어 있는지 식별이 필요
// 쿠키에 네트워크를 저장할 생각중...
{
    "address":""
}

// output
// 토큰, 통화의 어떤 값을 가져올지 생각 다시한번 해봐야될거 같음
{
    "id":"",
    "symbol":"",
    "balance": 0,
}
```

### Token(토큰 잔액 불러오기2)

- 내가 가진 토큰들의 잔액 불러오기
    - 각 토큰의 CA를 이용해서 잔액을 불러온다.
    - 현재 접속 네트워크에 따라서 잔액을 요청이 가능한지 알아봐야한다
    - 토큰의 ca값과 address만을 이용하면 잔액을 가져오는 것이 가능

```json
// input
{
    "address":"",
    "myToken": [
        "ca1",
        "ca2",
        "ca3",
        ...
    ]
}
```

### Token(토큰 추가기능)

- 커스텀 토큰 추가하기
    - ERC20의 검증을 진행하고 내 토큰 목록 myToken 혹은 tokenList에 추가
    - 토큰의 balance는 위에 있는 라우터에서 요청
    - 이미지를 어떻게 처리할지 결정해야함...

```json
// input
{
    "contractAddress":"",
    "symbol":"",
    "decimal":0
}

// output
// erc-20 토큰의 검증만 진행
{
    "success": true,?
}
```
